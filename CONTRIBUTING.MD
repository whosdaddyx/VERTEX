# Contributing to VERTEX

First of all, thank you for your interest in VERTEX! ❤️

While the core platform is currently closed source, there are still many ways you can contribute and help shape the future of the project.

---

# Ways to Contribute

## Report Bugs

Found an issue?

Please include:

* What happened
* What you expected to happen
* Steps to reproduce
* Screenshots (if applicable)
* Browser, operating system, and Discord client

The more information you provide, the faster we can investigate.

---

## Suggest Features

Have an idea that would improve VERTEX?

We love community feedback.

When suggesting a feature, try to explain:

* The problem you're trying to solve
* Your proposed solution
* Why it would benefit Discord communities

---

## Improve Documentation

Clear documentation is just as valuable as great code.

If you spot anything confusing, incomplete, or outdated, let us know.

---

## Test New Features

Community feedback is essential before major releases.

Testing early builds helps us improve stability, usability, and performance.

---

# Development Philosophy

VERTEX is built around a few core principles:

* Reliability first
* Security by design
* Performance matters
* User experience over feature count
* Continuous improvement

Every feature should solve a real problem—not simply add complexity.

---

# Community

Whether you report a bug, suggest a feature, or simply share feedback, you're helping make VERTEX better for everyone.

Thank you for being part of the journey. 🚀
