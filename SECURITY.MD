# Security Policy

At VERTEX, security is a core part of the platform.

We appreciate responsible disclosure of security vulnerabilities and will investigate all legitimate reports as quickly as possible.

---

## Supported Versions

Security updates are provided for the latest production release of VERTEX.

Older versions may not receive patches or security fixes.

---

## Reporting a Vulnerability

If you believe you've discovered a security vulnerability, please do **not** create a public GitHub issue.

Instead, contact us privately with as much information as possible, including:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Screenshots or proof of concept (if applicable)

---

## Responsible Disclosure

We kindly ask that you:

* Allow us reasonable time to investigate and resolve the issue.
* Avoid publicly disclosing vulnerabilities until a fix has been released.
* Act in good faith and avoid actions that could impact users or service availability.

---

## Security Principles

VERTEX is built with security as a priority.

Our platform follows industry best practices including:

* Secure authentication
* Role-based access control
* Input validation and sanitization
* Rate limiting
* Principle of least privilege
* Secure secret management
* Continuous dependency updates
* Regular security reviews

---

## Thank You

We appreciate everyone who helps improve the security of VERTEX through responsible disclosure.
